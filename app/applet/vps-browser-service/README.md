# Развертывание удаленных браузеров на РФ сервере

Эта папка содержит всё необходимое для запуска пула изолированных браузеров с доступом через noVNC (идеально для риелторов, чтобы они видели только сайт, а не DevTools интерфейс).

## Как запустить на VPS

1. Скопируй эту папку на свой РФ сервер (например, через `scp` или `git`).
2. Убедись, что на сервере установлен **Docker** и **Node.js** (версия 18+).
3. Перейди в терминале в эту папку:
   ```bash
   cd vps-browser-service
   ```

4. Собери Docker-образ с браузером:
   ```bash
   cd image
   docker build -t remote-browser:latest .
   cd ..
   ```

5. Открой файл `orchestrator.js` в любом редакторе (nano / vim) и замени `ТУТ_IP_ТВОЕГО_РФ_СЕРВЕРА` на реальный IP-адрес твоего сервера.

6. Установи зависимости для оркестратора:
   ```bash
   npm install
   ```

7. Запусти оркестратор (лучше использовать `pm2`, чтобы он работал в фоне):
   ```bash
   npm install -g pm2
   pm2 start orchestrator.js --name "browser-orchestrator"
   ```

## Важные моменты:
- Оркестратор слушает **порт 8080**.
- Оркестратор динамически открывает порты из диапазона **6000-7000** для VNC трансляций.
- Убедись, что эти порты (8080 и 6000-7000) открыты в Firewall (UFW / iptables) на твоем сервере.

## Как связать с AI Studio (Mini App)
Когда ты запустишь этот код, скопируй IP-адрес твоего сервера и отправь мне (агенту AI Studio), сказав: 
_"Я запустил оркестратор на IP XXX.XXX.XXX.XXX, подключай Mini App к нему."_

Я обновлю бэкенд в AI Studio, чтобы он начал делать `POST http://<ТВОЙ_IP>:8080/start-session` вместо Browserless.
